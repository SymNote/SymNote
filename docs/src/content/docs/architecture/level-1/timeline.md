Level 1 is the Global Context of your `.symnote` file. It acts as the "Master Timeline" of your song. 

Unlike Level 3 (where time flows microsecond by microsecond), time in Level 1 flows **block by block**. The engine reads your script from top to bottom, loading resources, defining tracks, and finally placing those tracks onto the timeline to be played.

---

## 1. Global Configuration & Resources

Before any music can be arranged, you must set the stage. The top of your file is dedicated to global setup.

### Setting the Tempo
The `set_bpm()` function is mandatory. It tells the engine how to convert musical grid fractions (like `1/16`) into exact milliseconds.
* *Example:* `set_bpm(130);`

### Loading Instruments
Audio files and synths must be loaded into memory before tracks can use them. Loading them at Level 1 ensures the engine allocates the memory only once.
* *Example:* `sample Kick = load_sample("kick.wav");`
* *Example:* `synth Lead = load_synth("sawtooth");`

### Defining Routines
Mathematical and logical functions (`routine`) must be defined globally so that any track can access them.

```ts
routine calc_swing(int step) returns float {
    return 0.05;
}
```

## 2. Track Execution (The Master Clock)

Once your `track` templates are defined (Level 2), you need to actually call them to generate music. 

When you call a track, the Master Clock **stops and waits** for that track to finish generating its timeline before moving to the next line of code. 

### What are Track Arguments? (Time vs. Parameters)
When you see a call like `Intro(4)`, it is vital to understand that **the engine does not know what a "bar" is.** The number `4` is simply an integer parameter passed into the track's Level 2 logic. The Master Timeline does not wait "4 bars"; it waits for the **total time generated by the grids inside that track**. We often use arguments like `4` conceptually to mean "bars" so our Level 2 loops are easier to write, but to the engine, it is just a number.

### Sequential Execution
Calling tracks one after another plays them sequentially.

```ts
// The timeline plays the Intro, waits for it to finish,
// then plays the BuildUp.
Intro(4);
BuildUp(4);
```

## 3. Concurrent Execution (`parallel`)

Music rarely consists of just one instrument playing at a time. To make multiple tracks play together, SymNote uses the `parallel` block.

When the engine hits a `parallel` block:
1. It pauses the Master Timeline.
2. It spawns independent execution threads for every track inside the block.
3. It waits for the **longest track** to finish.
4. Once the longest track is done, the Master Timeline resumes.

```ts
parallel {
    DrumMachine(4);
    Bassline(4);
    AmbientPad(8);
}
// The timeline will wait for AmbientPad to finish, 
// because its internal grids generate the longest duration of time.
```

## 4. Arrangement Loops (`loop` and `while`)

You can use both `loop` and `while` statements at Level 1 to easily structure your song without copying and pasting massive blocks of code. This is how you arrange repetitive sections like a Chorus or a Verse.

### Using `loop`
Perfect for a set number of repetitions.

```ts
// Play the Chorus section twice
loop (int repetition from 1 to 2) {
    parallel {
        DrumMachine(4);
        Bassline(4);
        LeadSynth(4);
    }
}
```

### Using `while`
Perfect for generative arrangements or conditional song structures.

```ts
int intensity = 1;

while (intensity <= 3) {
    parallel {
        DrumMachine(intensity * 2); // Pass dynamic parameters to the track
        Bassline(4);
    }
    intensity = intensity + 1;
}

// Then smoothly transition into the Outro
Outro(4);
```

## 5. Strict Level 1 Rules

Because Level 1 is the highest authority in the engine, it has a few strict rules to maintain architectural sanity:

* **No direct music generation:** You **cannot** write a `grid` block directly in Level 1. Music must be generated inside a `track`.
* **Global variables are permanent:** Variables declared in Level 1 (e.g., `bool play_drums = true;`) exist for the entire duration of the compilation. They do not get destroyed until the song is fully rendered.
* **Routine parameters:** Routines declared in Level 1 operate on a strictly "pass-by-value" basis. They cannot permanently modify global variables; they can only return new values.
