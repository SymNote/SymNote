Welcome to the **SymNote** technical hub. This space is dedicated to the clear, precise documentation of the language's mechanics.

This documentation is structured to give you a complete understanding of how the engine works, from the highest architectural concepts down to the smallest syntax rules.

---

### 📚 What you'll find here

To keep things organized, the documentation is split into three main conceptual areas:

* **🏛️ Core Architecture:** The fundamental philosophy behind SymNote. Learn about the overarching rules, how the AST processes logic, and the mechanics of the deterministic audio timeline.
* **🎹 The 3 Levels:** Deep dives into the specific environments of the language: the Master Timeline (Level 1), Track Logic (Level 2), and the Piano Roll Grid (Level 3).
* **📖 Reference:** Your technical dictionary. This covers static data structures, variable scoping, reserved keywords, and operator precedence.

---

### 🔍 How to navigate
Every page is designed to be a standalone reference. Use the **Sidebar** to browse by category, or press `Ctrl + K` (or `Cmd + K` on Mac) to instantly search for a specific keyword, rule, or concept.

> **Status:** This documentation is a living reflection of the SymNote source code. As the core logic evolves in the Java engine, these pages are updated to ensure absolute accuracy.
